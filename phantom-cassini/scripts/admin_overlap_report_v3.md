# 專案辦公室：行政工作疊加與衝突分析報告 (Workload Overlap & Conflict Analysis) V3

## 📌 關鍵發現：三大工作重疊期 (Major Bottlenecks)

本報告基於 **2059 份完整文件** 的全域掃描數據，證實了專案辦公室面臨嚴峻的多年度計畫疊加問題。
最嚴重的行政衝突發生在 **114年 Q2 (3-6月)**。

### 🚨 1. 紅色警戒區：114年 3月 (The Coaching Tsunami)
**衝突點**：同時處理 **「114年計畫的期初輔導」** 與 **「113年計畫的期末結案」**。

- **數據佐證**：
    - **Coaching: 67** (114 期初輔導工作坊)
    - **Review: 22** (113 期末審查/114 期初審查)
    - **Visit: 30** (113 期末查核)
    - **總計**：單月需處理 **119+** 份核心文件，是全年度最高峰。行政人員需同時面對縣市的期初提問與期末核銷。

### 🚨 2. 紅色警戒區：114年 5-6月 (The Documentation Hell)
**衝突點**：同時處理 **「114年計畫的修正計畫書審查」** 與 **「113年計畫的實地訪視」**。

- **數據佐證 (5月)**：
    - **Review: 34** (114 新案修正/核定)
    - **Visit: 13** (113 三書比對/訪視)
- **數據佐證 (6月)**：
    - **Review: 17** (114 核定持續)
    - **Visit: 27** (113 訪視高峰)
- **影響**：這兩個月是「審查」與「訪視」兩大文書魔王同時降臨的時刻。需同時準備大量委員評分表 (新案) 與 訪視紀錄/三書比對 (舊案)。

### ⚠️ 3. 潛在瓶頸：113年 4月 (Early Warning)
- **衝突點**：**Briefing: 4** (113 說明會) + **Visit: 4** (112 訪視)。雖然量能不如 114 年恐怖，但已出現「一邊辦活動、一邊跑訪視」的人力分流問題。

## 🛠️ 優化策略：針對重疊期的解法

針對上述發現的「3月海嘯」與「5-6月地獄」，建議優先開發以下 Skill：

1.  **AI 自動化三書比對 (Automated Compliance Check)**
    - **目標**：解決 5-6月 的 **Visit/Review 雙重文書負擔**。
    - **功能**：自動比對「計畫書」與「服務建議書」的經費與甘特圖差異。
    - **優先級**：⭐⭐⭐⭐⭐ (最高)

2.  **期初輔導問答機器人 (Coaching Q&A Bot)**
    - **目標**：解決 3月 的 **Coach: 67** 輔導海嘯。
    - **功能**：自動回答經費編列常見問題。
    *   **優先級**：⭐⭐⭐⭐⭐ (最高)

3.  **訪視會議紀錄生成器 (Visit Report Generator)**
    - **目標**：應對 6月 的 **Visit: 27** 訪視高峰。
    - **功能**：自動生成訪視紀錄表。
    - **優先級**：⭐⭐⭐⭐ (次高)
