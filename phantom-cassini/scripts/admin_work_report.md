# 專案辦公室行政工作量能分析報告 (111-114年)

## 📌 1. 年度工作重點概況 (Yearly Workload Overview)

本報告基於 **950+ 份文件** 的行政類型分類，分析專案辦公室在各年度的工作重心與行政負荷。

| 年度 | 總文件量 | 核心工作項目 (佔比) | 關鍵行政特徵 |
| :--- | :--- | :--- | :--- |
| **114年** | **485** 份 | **執行輔導 (37%)**, **審查核定 (27%)**, **徵件 (13%)** | 文件量最大，高度集中在 **3-5月** (徵件/審查/輔導同時進行)。 |
| **113年** | **277** 份 | **執行輔導 (35%)**, **審查核定 (17%)**, **徵件 (14%)** | 工作分佈較平均，**3月** 為年度高峰 (徵件/期初輔導)。 |
| **112年** | **175** 份 | **執行輔導 (32%)**, **審查核定 (23%)**, **行政庶務 (14%)** | 下半年 (9月) 活動較多，行政庶務與雜項文件佔比較高。 |
| **111年** | **21** 份 | **結案核銷 (52%)**, **審查 (38%)** | 以前導計畫為主，文件多為成果報告與審查紀錄。(樣本較少) |

## ⏳ 2. 專案生命週期與月份高峰 (Workflow Timeline)

透過月份熱點分析，可歸納出專案辦公室的標準作業時程 (SOP)：

### **Q1 (1-3月)：徵件與期初輔導 (High Intensity)**
- **重點工作**：計畫徵件說明會、申請須知公告、期初輔導會議。
- **數據佐證**：114年 **2月** 湧現大量「徵件」文件 (23份)；113-114年 **3月** 為「執行輔導」文件高峰 (10-94份)，顯示期初工作坊密集召開。

### **Q2 (4-6月)：審查與核定 (Critical Path)**
- **重點工作**：初審/複審會議、意見回復、核定計畫書修正。
- **數據佐證**：114年 **5月** 為「審查」文件高峰 (34份)，顯示此時正進行密集的方案評選與經費核定。

### **Q3 (7-9月)：執行訪視與管考 (Execution)**
- **重點工作**：期中查核、實地訪視、進度月報審閱。
- **數據佐證**：各年度此階段文件量平穩，以「輔導/訪視」類別為主。

### **Q4 (10-12月)：結案與成果驗收 (Closing)**
- **重點工作**：期末報告繳交、成果發表會、經費核銷。
- **數據佐證**：111年與112年資料顯示，**9-11月** 出現較多「結案」與「行政」類文件。

## 🛠️ 3. 未來優化建議 (Optimization Plan)

基於上述工作量分析，提出以下優化規劃：

1.  **Q1 徵件期自動化**：
    - 既然 2-3 月是文件高峰，建議導入 **「徵件文件自動檢查工具」**，自動掃描申請單位的計畫書缺漏，減輕人工初篩負擔。
2.  **Q2 審查期輔助**：
    - 針對 5 月的審查高峰，開發 **「審查意見自動彙整 Skill」**，將委員的 DOCX/PDF 意見表自動提取並產出 Excel 彙整表。
3.  **Q3 管考期追蹤**：
    - 建立 **「月報表自動讀取」** 機制，自動抓取各縣市月報中的關鍵進度與經費執行率，產出紅綠燈警示儀表板。
